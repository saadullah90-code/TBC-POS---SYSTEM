You are a senior full-stack engineer. Build a complete Production-Ready Retail POS + Inventory + Barcode System.

TECH STACK (IMPORTANT):
- Frontend: Next.js (React framework)
- Backend: Supabase (PostgreSQL + Auth + Storage)
- Database: Supabase (NOT MongoDB)
- Authentication: Supabase Auth (role-based)
- Barcode generation: bwip-js or equivalent
- Printing: window.print() with CSS print templates

---

PROJECT GOAL:
A professional multi-role retail system for shop inventory, barcode printing, POS billing, and real-time sales tracking.

---

ROLES SYSTEM:

1. ADMIN
- Full system control
- Dashboard analytics
- View sales (daily/weekly/monthly filters)
- Manage users (cashier, inventory staff)
- Full product + sales access

2. CASHIER
- POS billing screen only
- Scan barcode → add to cart
- Complete sale
- Print invoice
- Cannot edit inventory

3. INVENTORY STAFF
- Can add/edit products only
- Can generate and print barcodes
- Cannot access sales dashboard

---

Tables:

Users:
- id
- name
- email
- role (admin/cashier/inventory)

Products:
- id
- name
- title
- price
- category
- stock
- barcode (unique)
- created_at

Sales:
- id
- items (json array)
- total_amount
- cashier_id
- created_at

  - Add to cart

  - Remove items
- Total calculation
- Complete Sale button:
  - Save sale in Supabase
  - Reduce stock in Products table

- Only reduce stock on "Complete Sale"
